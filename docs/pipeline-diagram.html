Post-Pipeline
↻
/auto Improvement Loop
30-45 min per iteration
Run /auto N on a completed paper. Each iteration autonomously identifies the highest-impact improvements, executes them, and verifies no regressions. Content removal is a first-class improvement: cutting weak material strengthens the paper.
N iterations (user-specified) with early stop
Phase 1: Assess
5 parallel agents: depth/evidence, structure/flow, competitive positioning, writing quality, + Codex review. Regression detector on iterations 2+.
Phase 2: Prioritize
Reads assessments, ranks by impact. Regressions first, then WEAK/CRITICAL claims. Builds top-5 action plan. Early stop if <3 actions found.
Phase 3: Execute
Targeted research (max 3 queries), revision agent implements action plan. Cuts archived before deletion. Knowledge graph updated.
Phase 4: Verify
Compile, cross-ref check, no em dashes, recompute evidence scores. Cumulative context maintained for iterations 3+.
Provenance System
Full Provenance Chain
Final PDF paragraph ← provenance.jsonl entry (stage, agent, action, reasoning) ← research/sources/key.md (content snapshot of what was actually read) ← research/log.md (tool, query, URLs) ← Original paper (DOI/URL)
Every revision, cut, and expansion is also traced. Cut content is archived to provenance/cuts/ so nothing is silently lost.
